# ユーザーガイド

## 1. これは何ですか

SDTM Pedia は CDISC SDTM のための AI 支援ナレッジベースです。SDTMIG、SDTM Model、CDISC Controlled Terminology のうち、よく参照される内容を質問応答形式で確認できるように整理しています。

会話できる SDTM リファレンスとして利用できます。「この変数は何を意味するか」「この場面はどのドメインに入るか」「この統制用語は適用されるか」「これは SDTM 標準変数か」といった質問に向いています。

## 2. 想定ユーザー

- 臨床データマネジメント、統計プログラミング、標準管理、医学データレビュー担当者。
- SDTM ドメイン、変数、統制用語を確認したいプロジェクトメンバー。
- SDTM マッピング案を初期確認するチーム。
- SDTM の基本的な検索方法を説明または研修したいユーザー。

SDTM に不慣れな場合は、例示質問から始めてください。すでに SDTM に慣れている場合は、根拠確認付きの検索支援として利用できます。

## 3. 向いている質問

| 場面 | 例 |
| --- | --- |
| 変数定義 | `AESER とは何ですか。Core 属性は何ですか。` |
| ドメイン境界 | `LB、MB、IS はどう区別しますか。` |
| 統制用語 | `LBNRIND の submission values は何ですか。` |
| クロスドメイン関係 | `死亡は AE と DS でどのように表現しますか。` |
| 前提確認 | `SUPPTS は SDTMIG v3.4 で定義されていますか。` |

中国語、英語、日本語で質問できます。変数名、ドメインコード、C-code、submission value は英語コードのまま残すと、CDISC 資料との照合がしやすくなります。

## 4. どのプラットフォームを使うか

| 目的 | 推奨プラットフォーム | 理由 |
| --- | --- | --- |
| 複雑な標準解釈やクロスドメイン説明 | Claude Projects | 長めの説明と多段階の解釈に向いています。 |
| チーム共有と日常的な検索 | ChatGPT GPTs | 多くの組織で使いやすい入口です。 |
| 長い文脈の総合と広い比較 | Gemini Gems | 広い範囲の資料をまとめる場面に向いています。 |
| 厳格な出典境界と引用確認 | NotebookLM | アップロード済み資料に近い回答を求める場面に向いています。 |

通常はチームが提供する既定の入口から始めてください。管理者向けガイドは、インスタンスを設定、保守、検証する必要がある場合に参照します。

## 5. 回答をどう判断するか

業務参考として使いやすい回答は、通常以下を満たします。

- 関連する SDTM ドメイン、変数、統制用語、標準上の経路を明示する。
- 不適用または存在しない前提に対して、根拠なく補完しない。
- SDTMIG セクション、変数名、C-code、出典メモなど確認可能な根拠を示す。
- 現在のナレッジベース範囲外の質問では、その境界を明示する。

規制当局提出、重要なマッピング判断、正式成果物に関わる場合は、回答を出発点として、公式資料と内部レビューで確認してください。

## 6. クイックスタート

最初に以下を試してください。

1. `AESER とは何ですか。Core 属性と統制用語は何ですか。`
2. `LB、MB、IS はどう違いますか。微生物培養結果はどのドメインを検討しますか。`
3. `SUPPTS は SDTMIG v3.4 で定義されていますか。TSVAL が長すぎる場合は何を検討しますか。`

その他の例は [例示質問](./DEMO_QUESTIONS.ja.md) を参照してください。

## 7. 利用範囲

SDTM Pedia は CDISC 公式刊行物、統制用語の公式ソース、医学的判断、統計プログラミングレビュー、内部 SOP を置き換えません。正式提出に関わる判断は、必ず公式標準と組織内プロセスで確認してください。
