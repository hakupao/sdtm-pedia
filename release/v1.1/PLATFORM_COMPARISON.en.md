# Platform Selection

SDTM Pedia can be used on several AI platforms. The main differences are access model, team sharing, citation style, and how conservatively each platform handles questions outside the knowledge base.

## Quick Recommendation

| If You Need | Start With |
| --- | --- |
| Complex variable judgment, cross-domain explanation, and multi-step reasoning | Claude Projects |
| Team sharing, daily lookup, and a familiar ChatGPT entry point | ChatGPT GPTs |
| Long-context synthesis and broader comparison | Gemini Gems |
| Strict source boundaries and easy citation review | NotebookLM |

## Platform Overview

| Platform | Best Fit | Notes |
| --- | --- | --- |
| Claude Projects | SDTM variables, domain boundaries, cross-domain relationships, and complex standards explanation. | Works best when a team maintains one stable instance. External users may need a separate setup. |
| ChatGPT GPTs | Everyday team lookup, training demonstrations, and common terminology questions. | Depends on the organization's GPT sharing settings. Public publication may require platform review. |
| Gemini Gems | Broader synthesis across a larger material set and exploratory comparison. | Configuration text should be kept intact; use a team-verified instance when possible. |
| NotebookLM | Strict source-bound answers, citation review, and conservative handling of out-of-scope questions. | More likely to say that a question cannot be answered from the provided sources. |

## How To Choose

If you are an end user, start with the default entry point your team provides. Switch platforms only when your workflow needs a specific strength, such as stricter citations, easier team sharing, or broader synthesis.

If you are an administrator, first maintain one stable, access-controlled, lightly verified instance. Multiple platforms are useful because they complement one another; users do not need to use all of them at once.

## Quality Context

This release has been checked with questions covering variable definitions, controlled terminology, domain boundaries, cross-domain relationships, and common false premises. Platforms will still differ in style and conservatism, so formal work should be verified against official standards and team procedures.
