# Claude Project 管理者向けデプロイガイド

本ガイドは、SDTM Pedia の Claude Project を自分で設定または保守する管理者向けです。通常のユーザーは、チームが設定済みの Project にアクセスするだけで十分です。

## このガイドを使う場面

- 複雑な SDTM 標準照会に向いた共有入口が必要な場合。
- Claude Projects 上で承認済みの SDTM Pedia インスタンスを維持したい場合。
- リリースパッケージの知識ファイルと Project instructions を Claude に追加する場合。

## 前提条件

- Claude Project を作成できるアカウントと組織権限。
- 本リリースパッケージの `system_prompt.md` と `uploads/` フォルダへのアクセス。
- 組織が該当資料を Claude で利用することを許可し、データセキュリティ要件を満たしていること。

## デプロイ手順

1. Claude で新しい Project を作成します。
2. `system_prompt.md` の全文を Project instructions に貼り付けます。
3. `uploads/` 内の 19 個の Markdown ファイルを Project knowledge にアップロードします。
4. Project を保存し、名称、説明、アクセス権限を設定します。
5. 標準的な質問を少数実行し、変数定義、ドメイン境界、クロスドメイン説明が期待どおり機能することを確認します。

## 推奨名称

推奨:

> SDTM Pedia - Internal Reference

説明文には、CDISC 公式刊行物や内部 SOP を置き換えるものではなく、SDTM 標準確認を支援するツールであることを明記してください。

## 基本確認

デプロイ後、以下を確認してください。

- AE ドメインの主要変数と使用属性を説明できる。
- LB、MB、IS などの一般的なドメイン境界を区別できる。
- 標準にない変数やデータセットに対して慎重に対応できる。
- 回答に確認可能な根拠を含められる。

回答が想定から外れる場合は、Project instructions が完全か、19 個の知識ファイルがすべてアップロードされているか、ファイル名が変更されていないかを先に確認してください。

## チーム共有

共有前に以下を確認してください。

- Project にアクセスできるユーザー。
- instructions または知識ファイルを変更できるユーザー。
- プロジェクト機密、患者レベル情報、未匿名化データの入力禁止。
- ユーザー向けの利用境界説明が必要か。

誤変更を避けるため、編集権限は少数の保守担当者に限定してください。

## 保守

- 正式インスタンスを更新する前に、管理者用テスト Project を作成します。
- 確認後、正式インスタンスの instructions または知識ファイルを置き換えます。
- 更新日、保守担当者、変更概要、基本確認結果を記録します。

## 利用範囲

Claude Project インスタンスは SDTM 照会と説明の支援ツールです。正式提出、医学コーディング、プロジェクトレベルのマッピング、品質管理は、責任ある担当者が組織内手順に従って確認してください。
