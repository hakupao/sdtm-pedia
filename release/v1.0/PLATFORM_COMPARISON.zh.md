# 平台选择

SDTM Pedia 可以在多个 AI 平台上使用。不同平台的差异主要体现在访问方式、团队共享、引用呈现和对超出知识库范围问题的处理方式。

## 快速建议

| 如果你需要 | 优先考虑 |
| --- | --- |
| 复杂变量判断、跨域解释、较完整的推理过程 | Claude Projects |
| 团队共享、日常问答、熟悉的 ChatGPT 入口 | ChatGPT GPTs |
| 长上下文综合、较开放的资料比较 | Gemini Gems |
| 严格基于已上传资料回答、便于逐条回查来源 | NotebookLM |

## 平台对比

| 平台 | 适合场景 | 注意事项 |
| --- | --- | --- |
| Claude Projects | SDTM 变量、域边界、跨域关系和较复杂的标准解释。 | 适合由团队维护一个稳定实例；外部用户可能需要单独配置。 |
| ChatGPT GPTs | 团队日常查询、培训演示、常见变量和术语检索。 | 依赖组织的 GPT 共享设置；公开发布可能需要平台审核。 |
| Gemini Gems | 较长资料范围的综合查询、跨主题比较和探索式问题。 | 对配置文本完整性较敏感；建议使用团队确认过的实例。 |
| NotebookLM | 需要严格来源边界、引用回查和谨慎拒答的场景。 | 对不在资料范围内的问题会更倾向于说明无法回答。 |

## 如何选择

如果你是普通用户，建议先使用团队已经提供的默认入口。只有当某个平台在你的工作流中更方便，或你特别需要严格引用、团队共享、长上下文等特性时，再切换平台。

如果你是管理员，建议优先保证一个稳定、权限清晰、经过基础验证的平台实例，再考虑多平台并行。多平台的价值在于互补，而不是要求每个用户同时使用所有平台。

## 质量理解

本版本已用覆盖变量定义、受控术语、域边界、跨域关系和常见错误前提的测试集进行验证。不同平台的回答风格和保守程度会有差异，因此正式工作中仍应回查官方标准，并按团队流程进行复核。
