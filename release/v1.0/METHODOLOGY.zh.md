# 构建方法与验证声明

本页说明 SDTM Pedia v1.0 的资料来源、验证原则和适用边界。它面向需要了解可信度基础的用户，而不是内部工程记录。

## 1. 资料来源

知识库内容围绕 CDISC SDTM 相关官方资料整理，包括 SDTMIG、SDTM Model 和 CDISC Controlled Terminology。涉及医学编码或外部术语体系的内容，应结合对应授权来源和组织流程确认。

本项目不重新发布 CDISC 官方标准原文，也不把第三方摘要作为权威来源。回答中如涉及正式标准判断，应回到官方出版物核对。

## 2. 构建原则

知识库构建遵循以下原则:

- 以官方标准为主要依据。
- 保留域、变量、受控术语和章节之间的可追溯关系。
- 对无法完整展开或需要外部实时查询的内容明确说明边界。
- 对常见错误前提保持谨慎，不鼓励无依据补全。
- 把用户可读说明与内部构建记录分开。

## 3. 验证方式

v1.0 使用覆盖多类问题的测试集进行验证，包括变量定义、Core 属性、受控术语、域边界、跨域关系和常见错误前提。验证的目的不是证明工具可以替代人工复核，而是确认它在常见 SDTM 查询场景中能够给出有依据、可回查、边界清晰的回答。

对于正式工作，建议采用“AI 辅助查询 + 人工标准复核”的方式:

1. 先用 SDTM Pedia 获取候选答案和依据。
2. 回查 CDISC 官方出版物、受控术语来源或项目标准。
3. 将最终判断纳入组织内部 QC、审阅和批准流程。

## 4. 适用范围

适合:

- SDTM 学习和培训。
- 数据映射前的标准查询。
- 变量、域和受控术语的初步核对。
- 跨域关系和常见边界问题的解释。

不适合单独用于:

- 法规提交前的最终标准判定。
- 医学编码最终裁决。
- 替代统计编程、数据管理或标准治理团队的正式复核。
- 判断发布后发生变化的外部资料。

## 5. 责任边界

SDTM Pedia 是辅助工具。用户应根据使用场景的风险等级决定复核深度。任何影响正式提交、患者安全、医学解释或监管沟通的内容，都应由具备相应职责和资质的人员按组织流程确认。
