```

## 输入 (tier=high)

- `evidence_v2/D1_domain_list.md` (最终域清单, 需从 "## 最终清单 (28 个域)" 代码块解析)
- `knowledge_base/domains/<X>/examples.md` (每域一个文件)

推荐: 脚本支持**两种** --domain-list 输入格式:
1. 纯文本文件, 每行一个 domain abbr
2. D1_domain_list.md, 从 `## 最终清单` 下面第一个 ``` 代码块正则抓取域名

## 算法 (对每个高频域)

1. 读 `domains/<X>/examples.md`
2. 抽取顺序保留:
   - 域级标题 `# <X> — Examples` (在输出中改写为三级 `### <X> — Examples` 以统一)
   - 每个 Example 标题: `## Example N` → 改为 `#### Example N`
   - 每个 Example 下**首段 description** (≤ 2 行, 取第一个段落首 2 行), 不保留冗长业务背景段落
   - 粗体 XPT 文件名标记原样保留 (如 **ae.xpt**, **suppae.xpt**)
   - **Markdown 数据表**: 完整保留 (识别规则: 连续以 `|` 开头的行且包含 `|---|...|---|` 分隔行的块)
   - "shared with" / "see also" 交叉链接: 保留一行摘要
3. 跳过:
   - 除首段外的 description 段落
   - Business context narrative
   - `**Rows N-M:**` 解释行 (信息冗余, 表内可见)
4. 域间分隔:
   ```
   \n---\n
   <!-- source: knowledge_base/domains/<X>/examples.md -->
   ### <X> — Examples
   ```
5. 若域无 examples.md, 记录到 log, 输出 `### <X> — Examples\n(no examples file)` 占位

## 产出 (tier=high)

- `ai_platforms/claude_projects/output_v2/09_examples_data_high.md`
- 软目标 ≤ 30,000 tokens (cl100k_base)
- 硬上限 50,000 tokens — 超则 `sys.exit(1)` 并在 stderr 提示 "EXCEED_HARD_CAP"
- 第一行: `<!-- generated by scripts_v2/extract_examples_data.py --tier high at <ISO ts> -->`
- 末尾 stdout 打印:
  ```
  [Tier high] 09_examples_data_high.md: <N> tokens (target ≤30K, hard ≤50K)
  [Tier high] domains included: <count>
  [Tier high] domains missing examples.md: <list or "none">
  ```

## 强制要求

1. 源只读 (只 read mode, 不写 knowledge_base/)
2. 幂等 (两次运行 md5 同; 时间戳用 source mtime max 或文件列表 hash 派生, **不能用 wall-clock**)
3. Token 计算用 `tiktoken.get_encoding("cl100k_base")`
4. 所有路径用 `pathlib.Path`, REPO_ROOT 用 `Path(__file__).resolve().parents[3]`
5. Determinism: domain iteration 用 sorted, 表格行保序
6. 若 tier=others 被调用, exclude-list 格式: 每行一个 domain abbr (可注释 `#`), 排除这些, 剩余全部走
7. 若 token 落在 [30K, 50K]: 警告到 stderr 但不退出 (让主控决策)
