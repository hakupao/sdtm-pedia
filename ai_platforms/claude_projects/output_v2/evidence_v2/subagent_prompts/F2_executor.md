## Algorithm (tier=high)
For each C-code in the input list, in input order:
1. Locate the codelist section across ALL files under `knowledge_base/terminology/{core,questionnaires,supplementary}/`. Match by `## ... ({C-code})` heading.
2. If a codelist spans multiple files (e.g. general_part1 + general_part2 both have `## Epoch (C99079)`), aggregate all Term rows under the first-found heading; dedupe by Term `Code` (the row's leftmost C-code cell).
3. Preserve codelist heading + "Extensible: Yes/No" line + description if present.
4. Preserve Term table rows **verbatim** for Code / CDISC Submission Value / Synonyms cells.
5. **Truncate** the CDISC Definition cell to ≤200 characters (add `...` suffix if truncated). Keep `(NCI)` or other provenance tags inside the 200 char budget.
6. Append a 5th column "Related Domain" listing SDTM domains (sorted dir names) whose spec.md mentions this C-code. If none, write `—`.
7. Drop any "NCI Concept Description" expanded prose if present in source (not present in sampled file, but defensive).
