### §N26 Page-boundary off-by-one detection (NEW Hook 21, codifying round 12 batch 49 page-label Option H precedent)

**Source**: round 12 batch 49 page-label correction Option H 13 atoms (page off-by-one boundary p.22→p.23 + p.23→p.24 wrap detection). NEW pattern observation: page-boundary off-by-one motif on dense spec-table content where row continues across page footer/header. Writer's per-page extraction can mis-attribute atom to wrong physical page when row wraps.

**NEW Hook 21 (Self-Validate pre-DONE, post-extraction validation)**:

```python
# v1.8 N26 page-boundary off-by-one detection (Hook 21)
# Cross-page row physical-page disambiguation via pdftotext per-page extraction
# OR explicit footer 'Page N' marker tracking (sv20 has 'Page N' footers; ig34 lacks explicit Page N footers but page numbers visible in PDF metadata)

def hook_21_page_boundary_check(atom, pdf_text_by_page, page_index_authority):
    """
    For each TABLE_ROW / SENTENCE atom that may span page boundary:
    - Extract atom's verbatim
    - Check whether verbatim head matches end of page N OR start of page N+1
    - If verbatim is found in page N+1 only (not page N), but atom_id encodes page N → WARN page-boundary off-by-one motif
    - Apply page-label Option H if motif confirmed (writer-stage canonicalization, content unchanged)
    """
    page_from_atom_id = parse_page_from_atom_id(atom['atom_id'])
    verbatim_head = atom['verbatim'][:50]
    if verbatim_head not in pdf_text_by_page.get(page_from_atom_id, ''):
        if verbatim_head in pdf_text_by_page.get(page_from_atom_id + 1, ''):
            return ('WARN', f'page-boundary off-by-one detected: atom_id encodes p.{page_from_atom_id} but verbatim found in p.{page_from_atom_id+1}; recommend Option H page-label correction')
    return ('PASS', None)
```

**Trigger conditions**:
- Dense spec-table content (TABLE_ROW homogeneous) where row wraps page boundary
- Multi-row sustained-content-narrative across 5+ page sub-batch (e.g., round 12 batch 49 §3.1.3 Findings spec table p.20-29)
- Page-boundary motif on PDF sources with explicit footer 'Page N' marker (sv20) — Hook 21 check is high-confidence
- Page-boundary motif on PDF sources without explicit footer (ig34) — Hook 21 check via pdftotext per-page extraction comparison

**Rule v1.8 N26**: future round 13+ batches with dense spec-table content MUST run Hook 21 at writer-stage pre-DONE. Halt threshold: WARN-mode (non-blocking; logs to evidence; recommends Option H page-label correction); promote to halt-on-violation if motif persists at >1 atom per batch. Self-Validate hooks 20→21.
