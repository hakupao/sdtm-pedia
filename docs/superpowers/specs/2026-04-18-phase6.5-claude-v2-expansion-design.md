## 1. 背景与目标

### 1.1 触发

v1 完成上传后, Claude.ai UI 显示 **"12% of project capacity used"**, 而预期为 95-98%。`capacity_research.md` 已确认 200K 不是硬约束, paid 套餐 RAG 自动扩容到 ~3-4M tokens, v1 仅占用 RAG 容量的约 ~6-12%。

v1 (192K) 在 Smoke + T1-T8 全 PASS, 但 T3/T6/T7 出现"间接重建 / 边界模板兜底"场景, 损失了原文级精确度。

### 1.2 目标 (战略型 / B 方案)

补充 v1 过度压缩造成的精确度缺口, 但**不展开 terminology Term 值** (留给 Phase 7 评估)。

**成功标准** (v2.3 终态):

1. v2 在 v1 全部 T1-T8 测试上不衰减 (≥ v1 PASS 数, 不出现新 FAIL)
2. T3/T6/T7 三题从"间接重建/兜底"升级为"原文精确"
3. v2 总 token ≤ 450K (RAG 容量利用率 ≤ 15%, 远低于 ~3-4M 上限)
4. v1 baseline 完整保留, 任意时刻可回退
5. RAG 检索质量衰减观察纳入文档, 为 Phase 7 提供资产

### 1.3 非目标

- 不重做 v1 的 9 个压缩文件 (除批 3 替换 02_chapters)
- 不展开 terminology Term 值 (留 Phase 7)
- 不动 knowledge_base/ 任何文件 (源只读, P5)
- 不替换 v1 Project (用户手动新建第二个 Project, v1 保留至少 4 周)

## 2. 范围 (三批扩容)

### 2.1 批 1: chapters 关键补全 (+30-50K 估)

**新增内容**:
- `ch06_dataset_metadata` 完整 (v1 仅在 02_chapters 中精简保留)
- `ch08_supp_qualifiers` §8.3 完整 (v1 精简过)

**理由**: T3 (PC↔PP RELREC 4 种方法) 需要 ch06 数据集层规则 + ch08 supp 关联规则的完整描述, v1 精简版在抽样测试中暴露"间接重建"。

**产物**: `output_v2/09_chapters_addendum.md` (单文件)

### 2.2 批 2: examples 高频域数据表 (+50-100K 估)

**圈定算法** (推荐 B方案 + 用户高频清单):

**必选 (用户高频, 20 域)**:
DM, SE, DS, BS, BE, MI, GF, PR, CM, EX, TU, TR, RS, SS, DD, LB, FA, CE, MH, SU

**B 方案打分补充** (脚本 `score_domains.py` 跑出, 取 top 5-8):

```
score = w1 * cross_ref_count   (VARIABLE_INDEX 中被引用次数)
      + w2 * examples_tokens   (该域 examples.md 文件 token 数)
      + w3 * t_test_hit        (T1-T8 已涉及域 +20 加成)
weights: w1=0.4, w2=0.4, w3=0.2 (归一化)
```

**预期补充域 (打分高、用户清单未含的)**:
- AE (T1/T2 直接涉及)
- PC, PP (T3 直接涉及)
- VS (Findings 高频, 跨域引用多)
- EG (高频 Findings)
- QS (Questionnaire 高频)
- IE (Interventions 互动多)

**最终列表 = 用户 20 + 打分 top N**, 总域数预计 25-28。

**算法**: 每域提取 examples.md 的"数据表"段落 (markdown table), 去除冗长叙述, 保留:
- Example 标题
- 数据表 (Variable Name | Value 双列)
- 关键 Description 注释 (非示例性叙述)

**产物**: `output_v2/10_examples_data.md` (软目标 ≤ 30K, 硬上限 50K, 超 50K 拆 a/b)

### 2.3 批 3: chapters 撤销精简 (+30-50K 估)

**操作**: 替换 v1 的 `02_chapters.md`, 撤销 v1 对 ch01/02/03/08/10 的精简。

**保留原则**:
- ch04 完整保留 (与 v1 一致)
- ch01/02/03/08/10 全部恢复完整 (包括 v1 删除的举例段落)
- 章节内 Specification 表格保留所有列

**产物**: `output_v2/02_chapters.md` (替换型, 软目标 ≤ 90K, 硬上限 110K)

### 3.4 System Prompt 修订要点 (`system_prompt_v2.md`)

相对 v1 system_prompt.md, 必须新增:

1. **文件优先级**: 当 09_chapters_addendum 与 02_chapters 内容重叠时, **以 09 为准**; 当 10_examples_data 与 07_examples_catalog 内容重叠时, **数据问题以 10 为准, 索引导航以 07 为准**
2. **新文件介绍** (3 段): 09/10 文件分别说明用途、覆盖范围、何时优先引用
3. **边界模板修订**: T6 (问 example 数据) 现在多数情况能命中 10_examples_data, 边界模板降级为"非高频域才回 catalog 路径"
4. **路由规则微调**: examples 数据查询 → 优先 10, 命中失败再 fall back 到 07

## 5. 验收标准

### 5.1 Layer 1: 自动检查 (脚本层)

| # | 检查项 | 标准 | 执行方式 |
|---|-------|------|---------|
| C1 | v1 baseline 完整 | output_v1_baseline/ md5 与 output/ 一致 | md5 对比 |
| C2 | v2 不变文件未被修改 | output_v2/ 中 00/01/03/04/05/06/07/08 与 v1 md5 一致 | md5 对比 |
| C3 | 09 chapters_addendum 覆盖 ch06 + ch08 §8.3 | grep `# Chapter 6` + `## 8.3` | grep 计数 |
| C4 | 10 examples_data 域覆盖 | 25-28 域全部出现 `### XX` | grep 计数 |
| C5 | 02_chapters_v2 ch04 仍完整 | 字符数 ≥ ch04 源文件 95% | 对比 |
| C6 | 02_chapters_v2 撤销精简 | ch01/02/03/08/10 字符数 ≥ 各源文件 90% | 对比 |
| C7 | 总 token ≤ 450K | build_v2.py 汇总 | 累加 |
| C8 | 单文件 token ≤ 50K (硬上限) | build_v2.py 报告每文件 | 检查 |
| C9 | 源路径标注完整 | 每个合并输出 `<!-- source: -->` 全覆盖 | grep |
| C10 | system_prompt_v2 含优先级规则 | grep "以 09 为准" + "以 10 为准" | grep |
| C11 | 文件数 ≤ 15 (软目标) | ls output_v2/*.md | wc -l |

### 5.2 Layer 2: 渐进 A/B 测试 (用户上传后)

#### 5.2.1 测试题集

**继承 v1 T1-T8** (回归):
- T1 变量精确查询
- T2 规则推导
- T3 跨域关联 ⚠️ 重点
- T4 反向查询
- T5 路由准确性
- T6 边界-examples ⚠️ 重点
- T7 边界-terminology
- T8 边界-不编造

**新增 T9-T12** (扩容覆盖):
- T9 (批 2) "DM 的 Example 1 数据表是什么?" → 期望直接给数据表
- T10 (批 2) "EX 的剂量调整 Example 数据怎么写?" → 期望给数据表
- T11 (批 3) "ch01 中关于 Standard Foundational Model 的完整定义?" → 期望给完整段落
- T12 (批 3) "ch08 §8.3 supp 命名规则的所有细节?" → 期望给完整规则

#### 5.2.2 A/B 测试矩阵

每题在 v1 / v2 各跑一次, 由用户记录:

```
| 题目 | v1 答案 | v2 答案 | 精度变化 | 备注 |
|------|--------|--------|---------|------|
| T1   | <粘贴> | <粘贴> | 持平/↑/↓ | ... |
| ...
```

#### 5.2.3 PASS 条件

- T1-T8: v2 不出现新 FAIL (允许 v2 持平 v1, 不允许变差)
- T3/T6: v2 必须从 v1 的"间接重建/兜底"升级为"原文精确" (Δ = ↑)
- T9-T12: v2 必须 PASS (v1 因为没数据自然 FAIL, 这是预期)
- **回退/归因判定** (任一题 v2 < v1 时):
  - 衰减 1 题: 主控写 `failures/regression_v2_X.md`, 调 `code-reviewer` 归因, 不立即回退
  - 衰减 ≥ 2 题: 立即回退到上一阶段 (用户在 v2 Project 删除本批新增文件), 不进入下批
  - T3/T6 期望升级未达成 (仍是 v1 水平): 不算 FAIL 但记录为 G (gap) 留给 Phase 7
