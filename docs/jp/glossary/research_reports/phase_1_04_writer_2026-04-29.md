## 主 session 逆質問 (確認できない事項)

writer 執行段階で以下の点が判断保留対象となった。reviewer / 用語監査担当への申送り候補.

1. **「数日 / 数秒」表現の精度**: NFR-05 で「出典 PDF の該当頁および節を数秒で照合できる構造」と記述したが、METHODOLOGY.md 原文では "within seconds" 表現を直訳した。日本語の規制系文書では「数秒で」がぼかし語と見なされる可能性あり。reviewer に文体妥当性確認依頼.

2. **「処理単位」採用の文脈整合**: term_mapping.yml は batch の adopted を「処理単位」とするが、文中 FR-03「全 11 処理単位に分割」では「ロット」「区分」「ブロック」など他訳語の方が日常感に近い可能性あり。reviewer に文脈整合確認依頼.

3. **「(estimated)」の括弧表記**: NFR-03 で `(estimated)` を英語のまま括弧書き保持した。METHODOLOGY.md 原文の機械可読ラベルそのものを規格名として扱ったため。日本語版で統一する場合「(推定値)」など代替候補あり.

4. **「審査独立性原則」未使用**: term_mapping.yml で Rule D の adopted は「審査独立性原則」だが、本書は NFR-02 で「作成者・承認者分離」を観点列に置き、規律名 (Rule D) を導入していない。後段 03 試験結果報告書で「審査独立性原則」を本格採用する想定で、04 では機能要件レベルの記述に留めた.

5. **C-04 既知の限界の量的記述**: KNOWN_LIMITATIONS.en.md 由来の数値 (LB Test Code C65047 の 2,536 用語 / QS 系 296 のコードリスト) を yml に転記したが、これは AI 平台展開固有の数値であり、04 要件定義書の制約として記述する適切性に揺れがある。reviewer 判断委ねる.
