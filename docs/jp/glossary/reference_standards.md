## 4. 採用根拠記入規則

`term_mapping.yml` の各エントリに以下形式で記入:

```yaml
- internal: <内部用語>
  candidates: [<候補1>, <候補2>, <候補3>]
  adopted: <採用語>
  reason: |
    (採用理由 + 不採用候補の理由)
  reference:
    - "規格名 §節番号 (発行年)"
    - "(URL は本ファイル参照)"
```

**禁則**:
- ❌ 本リスト未掲載の規格を出典として使えない
- ❌ Wikipedia / 個人ブログ / GPT 一発訳出典禁止
- ❌ 出典ゼロの語は採用禁止
