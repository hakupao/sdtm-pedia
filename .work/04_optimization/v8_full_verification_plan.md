## 通用变量的特殊处理

通用变量（24 个，出现在 2+ 域）在 INDEX 中只显示一行汇总。验证规则：
1. **域列表**: INDEX 声明的域集合 == spec.md 中实际出现该变量的域集合
2. **Label/Type**: 与任意一个域的 spec.md 匹配即 PASS（跨域 Label 可能有变体，如 NHOID）
3. **Role/Core**: 如果带星号(*)，表示跨域不一致，验证星号标注是否正确：
   - 有星号 → 确实存在跨域差异（至少 2 个域的值不同）
   - 无星号 → 所有域的值必须一致

## 专属变量的验证规则

专属变量（1499 个，仅出现在 1 个域）逐字段精确匹配：
- Label: INDEX == spec.md 的 `**Label:**` 字段
- Type: INDEX == spec.md 的 `**Type:**` 字段
- Role: INDEX == spec.md 的 `**Role:**` 字段
- Core: INDEX == spec.md 的 `**Core:**` 字段
- CT: INDEX == spec.md 的 `**Controlled Terms:**` 字段（空=`—`，多值用分号分隔）

## PASS 标准

- 1917/1917 条目全部字段匹配
- 通用变量星号标注全部正确
- 0 个字段不匹配
